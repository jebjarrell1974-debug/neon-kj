Overview

This project is a Raspberry Pi–based karaoke system designed to automate karaoke shows in bars or entertainment venues. The system should manage singer rotation fairly, maintain crowd energy throughout the night, and provide a mobile interface for singers.

Hardware Platform

Primary device: Raspberry Pi 5 (8GB)

Expected hardware configuration:

Raspberry Pi 5  
NVMe SSD for local karaoke library  
DAC HAT for audio output  
HDMI output for lyrics display  
WiFi or Ethernet connectivity  

The Raspberry Pi acts as a karaoke server.

Users connect via phones to interact with the system.

7. Singer Rotation Algorithm

The rotation system must balance:

New singers  
Returning singers  
Fair stage time  

The system should prevent long waits while allowing regular singers to perform multiple times.

Target goal:

New singers should typically sing within 30–40 minutes.

---

Algorithm: Virtual Stage Time Scheduler

Each singer has a variable called:

virtual_stage_time

When a singer performs:

virtual_stage_time += song_length_minutes

The next singer selected is the singer with the lowest virtual_stage_time.

Pseudo code:

for each singer in queue:
    priority = virtual_stage_time

next_singer = singer with lowest priority

---

New Singer Priority Boost

New singers receive an initial virtual time bonus.

Example:

new_singer_virtual_time = -10

This allows new singers to enter the rotation quickly.

After their first song they merge into the normal scheduling system.

---

Song Length Fairness

Long songs automatically increase virtual_stage_time.

Example:

Singer A sings a 8-minute song

virtual_stage_time += 8

This means Singer A waits longer before singing again.

---

Wait Time Estimation

The system estimates wait time using average song duration.

Formula:

estimated_wait_time =
(number_of_singers_ahead × average_song_length)

Example:

5 singers ahead  
average song length = 4 minutes

Estimated wait = 20 minutes

8. Crowd Energy Tracking

The system collects song and performance data.

Data stored:

Song BPM  
Song genre  
Song duration  
Singer ID  
Time performed  
Wait time  
Crowd reaction rating  

These values allow the system to estimate crowd energy levels.

---

Energy Score Calculation

Each song receives an energy rating.

Example scale:

1–3 slow ballads  
4–6 medium energy  
7–10 high energy sing-alongs  

Example energy formula:

energy_score =
(BPM_factor + crowd_response + song_popularity) / 3

---

Energy Curve Model

Nightlife typically follows an energy curve.

Warm-up phase  
Peak phase  
Cool-down phase  

The system should maintain the highest energy between:

10 PM and 2 AM.

During this period the system should encourage high-energy songs.

---

Energy Priority Modifier

During peak hours the rotation algorithm may apply small priority bonuses.

Example:

if current_time between 10pm and 2am:

    if song_energy > 7:
        priority -= 2

This slightly promotes energetic songs without overriding fairness.

---

Slow Song Limiter

To avoid multiple slow songs in a row:

if last_three_songs_energy < 4:
    apply_priority_bonus_to_high_energy_songs()

The objective is to create an intelligent karaoke system that acts as a full show manager.

The system should:

Manage fair singer rotation  
Maintain crowd energy throughout the night

The system should operate as a self-contained karaoke appliance running on Raspberry Pi hardware.
